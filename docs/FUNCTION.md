# FUNCTION.md

STRUCTURE의 식별자(ID) 기준으로 요소별 기능·동작을 작성한다.
작성 규칙은 [FUNCTION_SPEC.md](./specs/FUNCTION_SPEC.md)를 따른다.

---

## 기능 목록 (AirTheremin V2)

### 공통 / 레이아웃 및 제어부

| ID (STRUCTURE와 동일)      | 트리거                                             | 동작·결과                                                                                                                                                                                            |
| -------------------------- | -------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **btn-start-audio**        | 버튼 클릭 (Click)                                  | Web Audio API `AudioContext` 활성화(resume) 및 카메라 권한 획득 -> 웹캠 피드(`webcam-viewport`) 초기화 -> 버튼을 '연주 중' 상태로 비활성화 전환.                                                     |
| **btn-align-calibrate**    | 버튼 클릭 (Click)                                  | 3초 화면 카운트다운 트리거 -> 사용자가 연주 시작 자세로 고정 대기 -> n초 완료 후 양손의 X/Y 중심 및 기본 손 벌림(Hand Span) 픽셀 거리 측정 -> 이를 0점 기준선(Baseline)으로 전역 상태 메모리에 로드. |
| **select-scale-lock**      | 드롭다운 값 변경 (Change)                          | 선택된 스케일(Chromatic, Pentatonic, Diatonic, Whole-tone) 상태 갱신 -> `right-pitch-grid` 상의 건반 시각 레이아웃 재배치 및 주파수 스냅 인덱스 재계산.                                              |
| **btn-swap-mode**          | 버튼 클릭 (Click)                                  | `SwapState` 토글 -> `left-play-zone`과 `right-play-zone`의 가로 배치 순서(Flex order) 및 연주/이펙터 역할 역할을 즉시 반전 적용.                                                                     |
| **visual-neon-glow**       | 손 랜드마크가 극단 화각(상하좌우 끝 10% 영역) 진입 | 화면 가장자리에 붉은색/네온 색상의 Neon Edge Glow CSS 애니메이션 클래스 추가 -> 화각 안으로 정상 진입 시 클래스 제거.                                                                                |
| **visual-particle-canvas** | 손 움직임 및 오디오 오실레이터 출력 (Continuous)   | 손가락 끝 궤적을 따라 몽환적인 네온 잔상 파티클(Neon Tail) 드로잉 + Web Audio AnalyserNode 데이터 기반 실시간 오디오 파형(Oscilloscope) 캔버스 드로잉.                                               |

### 페이지·라우트별 (`/` 단일 메인 화면)

| ID                           | 트리거                                           | 동작·결과                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ---------------------------- | ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **left-play-zone** (이펙터)  | 왼손 영역 감지 및 랜드마크 움직임 (Continuous)   | **Z축(손바닥 크기) 변화**: 작아질수록(멀어질수록) 리버브 `wet` 및 딜레이 피드백 증가 -> `left-reverb-bar` 게이지 실시간 높이 갱신.<br>**Y축(세로) 변화**: 손이 올라갈수록 로우패스 필터의 컷오프(Frequency)를 대수적(Logarithmic)으로 최대 10,000Hz까지 개방 -> `left-filter-bar` 게이지 갱신.                                                                                                                                                      |
| **right-play-zone** (멜로디) | 오른손 영역 감지 및 랜드마크 움직임 (Continuous) | **X축(최우측 랜드마크 좌표) 변화**: One Euro Filter 통과 -> Hysteresis ±5% 연산 -> 선택 음계 주파수 연산 -> 0.06초 Glide(linearRampToValueAtTime) 보간을 통해 오실레이터 2개 주파수 갱신 -> `right-pitch-grid` 내 활성 음 음영 강조.<br>**Hand Span(엄지-새끼 픽셀 거리) 변화**: 정규화된 손 벌림 너비 계산 -> 쥐었을 때(Fist, Mute 임계값 미만) Gain 0(Mute) -> 펼칠수록 Gain 0 ~ 1 exponentialRamp 보간 조절 -> `right-volume-meter` 실시간 갱신. |
| **left-status**              | 왼손 영역 내 손의 유무 감지 (Change)             | 왼손 감지 시 '이펙터 작동 대기 (Active)' 표시 -> 미감지 시 '왼손 감지 안 됨 (Inactive)'으로 텍스트 및 불투명도 변경.                                                                                                                                                                                                                                                                                                                                |
| **right-status**             | 오른손 영역 내 손의 유무 감지 (Change)           | 오른손 감지 시 '연주 준비 완료 (Active)' 표시 -> 미감지 시 '오른손 감지 안 됨 (Inactive)'으로 텍스트 및 불투명도 변경.                                                                                                                                                                                                                                                                                                                              |
